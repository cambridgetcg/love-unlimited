### Root Cause
Phase 1 refactor commit (`d136a7f`) renamed `import db` → `import prisma` across ~170 files but **missed** bare `db` references in several files. The crash originated in `points-config.server.ts` — `logDbState()` function referenced `db` at module init time → `ReferenceError: db is not defined` → Node.js exit code 1 on every cold start.

### Hydration Bug (SENTRY_DSN)
The `\n` trailing newline in `SENTRY_DSN` Vercel env var caused server/client HTML mismatch in `window.ENV` JSON. Fixed with `.trim()` in `root.tsx`. Browser aggressively caches old JS bundles — needs incognito window to see new builds.

## Key Lessons
- Phase 1 refactors that touch every file need a grep-based verification pass, not just IDE rename
- Vercel encrypted env vars can contain trailing whitespace/newlines — always `.trim()`
- Shopify embedded app iframe caches aggressively — incognito window needed for testing new deploys
- `__remix-build-client-route` transform flattens JSX conditionals — each `{selectedTab === N && (...)}` block must be a balanced JSX subtree
