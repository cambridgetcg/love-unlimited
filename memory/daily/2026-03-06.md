## ToK Spec Session with Yu

Yu spoke with another founder about ToK (Tree of Knowledge) architecture. We built out the full spec together at `docs/tok-spec.md`.

### Decisions made:
1. **TDU schema** — 5 types (instruction-response, conversation, correction, grounding-fact, reasoning-chain)
2. **Stake economics** — base 1 ZRN, difficulty multipliers 1-5×, dual staking (submitters + reviewers)
3. **Simulation validated** — quality agents profit most, all dishonest strategies lose money
4. **Reviewer stake ratio: 0.3** — kills rubber-stamping and lazy-reject strategies
5. **Weighted reviews, not gated** — any agent can review anything, domain reputation = vote weight
6. **Cold start: stake-only** — no reputation requirements initially
7. **Protocol-gated licensing** — open data, ZRN for API access
8. **First vertical: code**
9. **Mixture fine-tuning** — 70% domain / 30% general for vertical models
10. **Living memory** — TDU fitness decay with feedback loop from model evaluation
11. **Dataset sharding** — validators hold unique shards, complete dataset only in TEE for training
12. **Dispute resolution** — default reject with grace, stakes returned, 3 strikes = permanent
13. **Fitness compute** — funded from API revenue, exact split TBD post-launch
