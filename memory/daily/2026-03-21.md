### Bug #1: get_market_trades used wrong API param
- Data API param is `market` (not `conditionId`) for filtering trades by market
- Using `conditionId` as param name → API silently ignores → returns global trade feed
- **All prior scan results were garbage** — same wallets across all markets because same data
- Fixed in insider.py

### Bug #2: get_user_trades used wrong API param  
- Data API param is `user` (not `proxyWallet`) for filtering trades by wallet
- Same silent-ignore pattern
- Profile command was returning global feed instead of wallet-specific trades
- Fixed in insider.py

### Root cause
Polymarket Data API silently ignores unknown query params and returns unfiltered results. No error, no warning. Extremely deceptive. Both bugs were present since tool creation (Mar 19).

### Key lesson
**Always verify API data belongs to the entity you requested.** The fix was literally checking `conditionId match: X/Y` in the response. Should have been in the original code.
