## SSH Blocker Investigation

**Investigated SSH key auth blocker for Sentry (root@135.181.28.252)**

### Findings:

1. **Local SSH keys:**
   - `~/.ssh/id_ed25519` exists (Ed25519 key)
   - `~/.ssh/id_rsa` does NOT exist (no RSA key)
   - `~/.ssh/hive-key` exists (Ed25519 key, authorized on Sentry)
   - `~/.ssh/hive-key.pub` exists (public key)

2. **SSH config for Sentry:**
   - Host `sentry` configured in `~/.ssh/config` with:
     - `HostName 135.181.28.252`
     - `User root`
     - `IdentityFile ~/.ssh/hive-key`
   - Using alias `ssh sentry` works perfectly

3. **Connection test results:**
   - `ssh sentry` → **SUCCESS** (uses hive-key from config)
   - `ssh root@135.181.28.252` → **FAILS** (tries default keys: id_rsa, id_ecdsa, id_ed25519)
   - `ssh -o IdentityFile=~/.ssh/hive-key root@135.181.28.252` → **SUCCESS**

4. **Root cause:**
   - Deployment scripts (e.g., `fleet-agent-deploy.sh`) use `root@135.181.28.252` directly
   - They don't specify `IdentityFile` in SSH_OPTS
   - SSH tries default keys (id_rsa, id_ecdsa, id_ed25519) but hive-key is not a default key
   - No RSA key exists locally, so RSA authentication fails

### What's needed to unblock provision.sh deployment:

**Option 1 (Recommended):** Update deployment scripts to use the SSH config alias:
- Change `root@135.181.28.252` to `sentry` in scripts
- Or add `-o IdentityFile=~/.ssh/hive-key` to SSH_OPTS

**Option 2:** Create an RSA key pair:
- Generate RSA key: `ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa`
- Add public key to Sentry: `ssh-copy-id -i ~/.ssh/id_rsa.pub sentry`
- This would allow default SSH behavior to work

**Option 3:** Add hive-key to SSH agent:
- `ssh-add ~/.ssh/hive-key`
- SSH agent would then offer this key during authentication

**Immediate fix:** The simplest is to modify `fleet-agent-deploy.sh` to include `-o IdentityFile=~/.ssh/hive-key` in its `SSH_OPTS` variable, or use the `sentry` alias instead of the raw IP address.

**Verification:** SSH connection with hive-key works, key is authorized on server, server accepts RSA keys but local RSA key doesn't exist.
